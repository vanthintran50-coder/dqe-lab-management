# DQE Lab Management System - Database Documentation\n\n## 📋 Database Overview\n\n**Database Name:** `DQELabDB`\n**DBMS:** SQL Server 2019+\n**Purpose:** Store and manage laboratory equipment, test requests, bookings, and user data\n\n---\n\n## 📊 Database Schema\n\n### 1. Users Table\nStores user information and authentication details.\n\n```sql\nCREATE TABLE Users (\n    Id INT PRIMARY KEY IDENTITY(1,1),\n    Name NVARCHAR(255) NOT NULL,\n    Email NVARCHAR(255) NOT NULL UNIQUE,\n    Department NVARCHAR(255),\n    Role NVARCHAR(50) DEFAULT 'user',  -- admin, manager, user\n    AzureId NVARCHAR(255),\n    IsActive BIT DEFAULT 1,\n    CreatedAt DATETIME DEFAULT GETDATE(),\n    UpdatedAt DATETIME DEFAULT GETDATE()\n);\n```\n\n**Columns:**\n- `Id` - Primary key\n- `Name` - User full name\n- `Email` - Unique email address\n- `Department` - User department\n- `Role` - User role (admin, manager, user)\n- `AzureId` - Microsoft Azure AD ID\n- `IsActive` - Account active status\n- `CreatedAt` - Record creation timestamp\n- `UpdatedAt` - Last update timestamp\n\n---\n\n### 2. Equipment Table\nStores laboratory equipment information and availability status.\n\n```sql\nCREATE TABLE Equipment (\n    Id INT PRIMARY KEY IDENTITY(1,1),\n    Name NVARCHAR(255) NOT NULL,\n    Model NVARCHAR(255) NOT NULL,\n    Location NVARCHAR(255),\n    Status NVARCHAR(50) DEFAULT 'available',\n    Description NVARCHAR(MAX),\n    Icon NVARCHAR(255),\n    CreatedAt DATETIME DEFAULT GETDATE(),\n    UpdatedAt DATETIME DEFAULT GETDATE()\n);\n```\n\n**Columns:**\n- `Name` - Equipment name\n- `Model` - Equipment model number\n- `Location` - Physical location\n- `Status` - Current status (available, in-use, maintenance, reserved)\n- `Description` - Equipment description\n- `Icon` - Equipment icon/emoji\n\n**Status Values:**\n- `available` - 可用\n- `in-use` - 使用中\n- `maintenance` - 維修中\n- `reserved` - 已預約\n\n---\n\n### 3. TestRequests Table\nStores test request submissions from users.\n\n```sql\nCREATE TABLE TestRequests (\n    Id INT PRIMARY KEY IDENTITY(1,1),\n    RequestNumber NVARCHAR(100) NOT NULL UNIQUE,\n    EquipmentId INT NOT NULL,\n    Customer NVARCHAR(255) NOT NULL,\n    TestPhase NVARCHAR(50) NOT NULL,  -- EVT, PTT, MP, ORT\n    EstimatedStartDate DATETIME NOT NULL,\n    EstimatedEndDate DATETIME NOT NULL,\n    RequesterId INT NOT NULL,\n    Requester NVARCHAR(255),\n    SampleCount INT DEFAULT 0,\n    Status NVARCHAR(50) DEFAULT 'open',\n    Description NVARCHAR(MAX),\n    RequestDate DATETIME DEFAULT GETDATE(),\n    CreatedAt DATETIME DEFAULT GETDATE(),\n    UpdatedAt DATETIME DEFAULT GETDATE(),\n    FOREIGN KEY (EquipmentId) REFERENCES Equipment(Id),\n    FOREIGN KEY (RequesterId) REFERENCES Users(Id)\n);\n```\n\n**Test Phases:**\n- `EVT` - Evaluation Test\n- `PTT` - Performance Test\n- `MP` - Mass Production\n- `ORT` - Ongoing Reliability Test\n\n**Status Values:**\n- `open` - 開放\n- `in-progress` - 進行中\n- `completed` - 已完成\n- `cancelled` - 已取消\n\n---\n\n### 4. Bookings Table\nStores equipment reservation/booking information.\n\n```sql\nCREATE TABLE Bookings (\n    Id INT PRIMARY KEY IDENTITY(1,1),\n    RequestId INT NOT NULL,\n    EquipmentId INT NOT NULL,\n    StartDate DATETIME NOT NULL,\n    EndDate DATETIME NOT NULL,\n    UserId INT NOT NULL,\n    Status NVARCHAR(50) DEFAULT 'pending',\n    Notes NVARCHAR(MAX),\n    CreatedAt DATETIME DEFAULT GETDATE(),\n    UpdatedAt DATETIME DEFAULT GETDATE(),\n    FOREIGN KEY (RequestId) REFERENCES TestRequests(Id),\n    FOREIGN KEY (EquipmentId) REFERENCES Equipment(Id),\n    FOREIGN KEY (UserId) REFERENCES Users(Id)\n);\n```\n\n---\n\n### 5. Approvals Table\nStores multi-level approval workflow information.\n\n```sql\nCREATE TABLE Approvals (\n    Id INT PRIMARY KEY IDENTITY(1,1),\n    RequestId INT NOT NULL,\n    ApprovalOrder INT,\n    ApproverId INT,\n    Status NVARCHAR(50) DEFAULT 'pending',  -- pending, approved, rejected\n    Comments NVARCHAR(MAX),\n    ApprovedAt DATETIME,\n    CreatedAt DATETIME DEFAULT GETDATE(),\n    FOREIGN KEY (RequestId) REFERENCES TestRequests(Id),\n    FOREIGN KEY (ApproverId) REFERENCES Users(Id)\n);\n```\n\n---\n\n### 6. Tasks Table\nStores task tracking and assignment information.\n\n```sql\nCREATE TABLE Tasks (\n    Id INT PRIMARY KEY IDENTITY(1,1),\n    BookingId INT NOT NULL,\n    Status NVARCHAR(50) DEFAULT 'pending',  -- pending, in-progress, completed\n    AssignedToId INT,\n    CreatedDate DATETIME DEFAULT GETDATE(),\n    CompletedDate DATETIME,\n    Notes NVARCHAR(MAX),\n    CreatedAt DATETIME DEFAULT GETDATE(),\n    UpdatedAt DATETIME DEFAULT GETDATE(),\n    FOREIGN KEY (BookingId) REFERENCES Bookings(Id),\n    FOREIGN KEY (AssignedToId) REFERENCES Users(Id)\n);\n```\n\n---\n\n### 7. AuditLogs Table\nStores system audit trail for compliance and monitoring.\n\n```sql\nCREATE TABLE AuditLogs (\n    Id INT PRIMARY KEY IDENTITY(1,1),\n    UserId INT,\n    Action NVARCHAR(255),\n    TableName NVARCHAR(255),\n    RecordId INT,\n    OldValue NVARCHAR(MAX),\n    NewValue NVARCHAR(MAX),\n    CreatedAt DATETIME DEFAULT GETDATE(),\n    FOREIGN KEY (UserId) REFERENCES Users(Id)\n);\n```\n\n---\n\n## 📈 Database Indexes\n\nThe following indexes are created for query performance optimization:\n\n```sql\nCREATE INDEX idx_equipment_status ON Equipment(Status);\nCREATE INDEX idx_testrequests_status ON TestRequests(Status);\nCREATE INDEX idx_testrequests_equipment ON TestRequests(EquipmentId);\nCREATE INDEX idx_bookings_equipment ON Bookings(EquipmentId);\nCREATE INDEX idx_bookings_startdate ON Bookings(StartDate);\nCREATE INDEX idx_tasks_status ON Tasks(Status);\n```\n\n---\n\n## 🔗 Entity Relationship Diagram\n\n```\nUsers\n  ├── TestRequests (RequesterId)\n  ├── Bookings (UserId)\n  ├── Approvals (ApproverId)\n  └── Tasks (AssignedToId)\n\nEquipment\n  ├── TestRequests (EquipmentId)\n  └── Bookings (EquipmentId)\n\nTestRequests\n  ├── Bookings (RequestId)\n  ├── Approvals (RequestId)\n  └── Tasks (via Bookings)\n\nBookings\n  └── Tasks (BookingId)\n\nAll tables\n  └── AuditLogs\n```\n\n---\n\n## 💾 Data Types Used\n\n- `INT` - Integer (auto-increment for IDs)\n- `NVARCHAR(255)` - Variable-length Unicode string (standard text)\n- `NVARCHAR(MAX)` - Variable-length Unicode string (long text)\n- `DATETIME` - Date and time\n- `BIT` - Boolean (0 or 1)\n\n---\n\n## 🔐 Security Considerations\n\n1. **Unique Email:** User email must be unique\n2. **Foreign Keys:** Maintain referential integrity\n3. **Timestamps:** Track record creation and updates\n4. **Audit Logs:** Monitor all changes for compliance\n5. **Soft Deletes:** Use `IsActive` flag instead of hard deletes\n\n---\n\n## 📊 Sample Queries\n\n### Get available equipment\n```sql\nSELECT * FROM Equipment WHERE Status = 'available';\n```\n\n### Get pending test requests\n```sql\nSELECT * FROM TestRequests WHERE Status = 'open';\n```\n\n### Get equipment booking schedule\n```sql\nSELECT e.Name, b.StartDate, b.EndDate, u.Name as BookedBy\nFROM Bookings b\nJOIN Equipment e ON b.EquipmentId = e.Id\nJOIN Users u ON b.UserId = u.Id\nWHERE b.StartDate >= GETDATE()\nORDER BY b.StartDate;\n```\n\n### Get user activity\n```sql\nSELECT u.Name, a.Action, a.TableName, a.CreatedAt\nFROM AuditLogs a\nJOIN Users u ON a.UserId = u.Id\nWHERE a.CreatedAt >= DATEADD(DAY, -7, GETDATE())\nORDER BY a.CreatedAt DESC;\n```\n\n---\n\n## 🔄 Backup & Recovery\n\n### Backup Database\n```sql\nBACKUP DATABASE DQELabDB\nTO DISK = 'C:\\Backups\\DQELabDB.bak'\nWITH INIT;\n```\n\n### Restore Database\n```sql\nRESTORE DATABASE DQELabDB\nFROM DISK = 'C:\\Backups\\DQELabDB.bak';\n```\n\n---\n\n**Last Updated:** 2026-07-18\n"
