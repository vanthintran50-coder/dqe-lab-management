# OCS DQE Lab Management System - Setup & Installation Guide\n\n## 📋 Prerequisites\n\n- **Node.js** v16 or higher\n- **npm** or **yarn**\n- **SQL Server** 2019 or higher (Local or Remote)\n- **Git**\n- **Microsoft Azure Account** (for Azure AD authentication)\n\n## 🚀 Quick Start\n\n### 1️⃣ Clone the Repository\n\n```bash\ngit clone https://github.com/vanthintran50-coder/dqe-lab-management.git\ncd dqe-lab-management\n```\n\n### 2️⃣ Setup Frontend\n\n```bash\ncd frontend\nnpm install\n```\n\nCreate `.env` file in the `frontend` directory:\n\n```env\nREACT_APP_CLIENT_ID=your-azure-app-id\nREACT_APP_AUTHORITY=https://login.microsoftonline.com/your-tenant-id\nREACT_APP_REDIRECT_URI=http://localhost:3000\nREACT_APP_API_URL=http://localhost:5000/api\n```\n\nStart the development server:\n\n```bash\nnpm start\n```\n\nThe frontend will be available at: **http://localhost:3000**\n\n### 3️⃣ Setup Backend\n\n```bash\ncd ../backend\nnpm install\n```\n\nCreate `.env` file in the `backend` directory:\n\n```env\nPORT=5000\nNODE_ENV=development\nDB_SERVER=localhost\nDB_NAME=DQELabDB\nDB_USER=sa\nDB_PASSWORD=your-password\nJWT_SECRET=your-secret-key\nAZURE_CLIENT_ID=your-azure-app-id\nAZURE_CLIENT_SECRET=your-azure-secret\nAZURE_TENANT_ID=your-tenant-id\nPOWERBI_WORKSPACE_ID=your-workspace-id\n```\n\nStart the backend server:\n\n```bash\nnpm run dev\n```\n\nThe backend API will be available at: **http://localhost:5000/api**\n\n### 4️⃣ Setup Database\n\nOpen **SQL Server Management Studio** (SSMS) and run:\n\n```bash\nsqlcmd -S localhost -U sa -P your-password -i database/schema.sql\n```\n\nOr directly in SSMS:\n\n1. Open `database/schema.sql`\n2. Connect to your SQL Server\n3. Execute the script\n\n## 🔐 Azure AD Configuration\n\n### Register Application\n\n1. Go to **Azure Portal** → **App registrations**\n2. Click **New registration**\n3. Name: `DQE Lab Management`\n4. Supported account types: `Single tenant`\n5. Redirect URI: `http://localhost:3000`\n6. Click **Register**\n\n### Get Credentials\n\n1. Go to **Overview** tab:\n   - Copy **Application (client) ID** → `REACT_APP_CLIENT_ID`\n   - Copy **Directory (tenant) ID** → `REACT_APP_AUTHORITY`\n\n2. Go to **Certificates & secrets**:\n   - Click **New client secret**\n   - Copy the value → `AZURE_CLIENT_SECRET`\n\n3. Go to **API permissions**:\n   - Add `User.Read` permission\n   - Grant admin consent\n\n## 📊 Power BI Integration\n\n1. Create a **Power BI workspace**\n2. Create reports and dashboards\n3. Get the **Workspace ID** and **Report ID**\n4. Update Admin Panel with these credentials\n5. Configure Power BI service principal (optional for production)\n\n## 🗄️ Database Tables\n\n- **Users**: User management and authentication\n- **Equipment**: Equipment inventory\n- **TestRequests**: Test request submissions\n- **Bookings**: Equipment reservations\n- **Approvals**: Approval workflow\n- **Tasks**: Task tracking\n- **AuditLogs**: System audit trail\n\n## ✨ Features\n\n✅ **Dashboard** - KPI metrics and overview\n✅ **Equipment Booking** - Reserve equipment with calendar\n✅ **Test Request Form** - Submit test requests\n✅ **Approval Workflow** - Multi-level approval process\n✅ **Task Management** - Kanban-style task board\n✅ **Equipment Management** - Manage equipment inventory\n✅ **Microsoft Login** - Azure AD authentication\n✅ **Power BI Integration** - Analytics and reporting\n✅ **SQL Server Database** - Robust data storage\n✅ **Admin Panel** - System configuration\n\n## 📁 Project Structure\n\n```\ndqe-lab-management/\n├── frontend/                 # Run production build\n```\n\n## 🐛 Troubleshooting\n\n### Port Already in Use\n\n**Windows:**\n```bash\nnetstat -ano | findstr :5000\ntaskkill /PID <PID> /F\n```\n\n**macOS/Linux:**\n```bash\nlsof -i :5000\nkill -9 <PID>\n```\n\n### Database Connection Failed\n\n- Verify SQL Server is running\n- Check credentials in `.env`\n- Ensure firewall allows SQL Server port (1433)\n- Check SQL Server authentication mode\n\n### Azure AD Login Not Working\n\n- Verify Client ID and Tenant ID\n- Check redirect URI matches Azure portal\n- Clear browser cache and cookies\n- Check Azure AD app permissions\n\n## 📧 Support\n\nFor issues or questions, please contact: **vanthintran50@gmail.com**\n\n## 📄 License\n\nMIT License - See LICENSE file for details\n\n---\n\n**Happy Coding! 🎉**\n
